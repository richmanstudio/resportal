# MVP Roadmap

Этот документ фиксирует путь от текущего прототипа РЕСПОРТАЛ до полноценного MVP, который можно отдавать реальным пользователям.

Продакшен-деплой намеренно вынесен в отдельный финальный этап и не входит в текущий рабочий scope. До него нужно довести продуктовую, техническую и юридическую готовность.

## Как Вести Roadmap

Статусы:

- `[ ]` не начато
- `[~]` в работе
- `[x]` готово
- `[!]` заблокировано или требует решения

Правило обновления: после каждой завершенной доработки отмечать выполненные пункты и добавлять короткую заметку в раздел "Журнал прогресса".

## Scope MVP

Целевой пользователь: частный юрист или небольшая юридическая фирма.

Основной пользовательский путь:

`регистрация -> организация -> клиенты -> дела -> стороны -> задачи и сроки -> календарь -> документы -> DOCX -> команда -> оплата`

Не входит в MVP:

- AI-функции
- интеграции с судебными системами
- мобильное приложение
- микросервисы
- сложная BI-аналитика
- production deployment, HTTPS, Nginx, backup/rollback инфраструктура
